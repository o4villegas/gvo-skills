# SEO Content Optimizer

## Overview
The SEO Content Optimizer skill helps marketers improve organic search visibility through data-driven content optimization. It combines keyword research, technical SEO implementation, and content strategy to increase rankings and qualified traffic.

## When to Use This Skill
- Creating new content that needs SEO optimization
- Auditing and improving existing underperforming content
- Targeting specific keywords or search intents
- Improving site structure and internal linking
- Writing or optimizing meta tags and snippets
- Planning content strategy around search demand

## Keyword Research Workflow

### Phase 1: Seed Keywords and Topic Clustering

**Step 1: Identify Seed Keywords**
- Core product/service keywords
- Brand-adjacent keywords
- Problem-solving queries
- Question-based keywords (how, why, what)

**Example:** For SaaS project management tool:
- "project management software"
- "team collaboration tool"
- "agile project management"
- "how to manage remote team projects"

**Step 2: Expand Keywords Using:**
- Google Search Console (query data)
- Google Suggest (autocomplete)
- Answer the Public (question queries)
- Competitor analysis tools
- Industry forums and communities

**Expanded List Template:**
| Keyword | Search Volume | Difficulty | Intent | Relevance | Priority |
|---------|---------------|------------|--------|-----------|----------|
| project management software | 12,100 | High | Commercial | Very High | High |
| how to manage projects remotely | 450 | Low | Informational | High | High |
| agile software tools comparison | 720 | Medium | Commercial | High | Medium |
| project timeline tracking | 280 | Low | Informational | Medium | Medium |

### Phase 2: Intent Classification

**Commercial Intent** (High conversion potential)
- Keywords: "best X for Y", "X software comparison", "X pricing"
- Content Type: Product pages, comparison guides
- Goal: Drive conversions/demos

**Informational Intent** (Build authority)
- Keywords: "how to X", "what is X", "guide to X"
- Content Type: Blog posts, tutorials, guides
- Goal: Build trust, capture early-funnel leads

**Navigational Intent** (Brand strength)
- Keywords: "[Brand] + feature", "[Brand] login"
- Content Type: Product docs, support pages
- Goal: Support existing customers

## On-Page SEO Implementation Checklist

### Page Title Optimization

**Guidelines:**
- Length: 50-60 characters (displays fully in most SERPs)
- Include primary keyword near the beginning
- Include brand name (if space permits)
- Make it compelling and click-worthy

**Template:**
```
[Primary Keyword] | [Benefit/Unique Angle] | [Brand Name]
```

**Examples:**
- ❌ "Project Management Software"
- ✓ "Project Management Software for Remote Teams | [Brand]"
- ❌ "Blog Post About Marketing"
- ✓ "Marketing Campaign Planning Guide: Strategies & Checklists"

### Meta Description Optimization

**Guidelines:**
- Length: 155-160 characters
- Include primary keyword naturally
- Include compelling call-to-action
- Avoid keyword stuffing
- Make unique for each page

**Template:**
```
[What page covers] + [Key benefit] + [CTA]
```

**Examples:**
- ❌ "This page is about project management software features."
- ✓ "Manage projects 40% faster with AI-powered planning tools. Free trial. See how 10K+ teams collaborate."

### Headers (H1, H2, H3) Strategy

**H1 (Main heading):**
- One H1 per page
- Include primary keyword
- Make descriptive and user-focused

**H2 (Subheadings):**
- Use for main sections
- Include semantic variations and related keywords
- Create logical content hierarchy

**H3 (Sub-subheadings):**
- Organize content within sections
- Support readability and skimmability

**Template:**
```
# [Primary Keyword] + [Value Proposition]

## What Is [Keyword]?
## How to [Action] with [Keyword]
## Best Practices for [Keyword]

### [Specific sub-topic 1]
### [Specific sub-topic 2]
```

### Image Optimization

**File Names:**
- Descriptive, keyword-relevant
- Use hyphens to separate words
- ✓ "project-management-team-dashboard.png"
- ❌ "image123.png"

**Alt Text:**
- Describe image content
- Include relevant keywords naturally
- 8-15 words typical length
- Example: "Team project dashboard showing timeline view with color-coded task assignments"

## Internal Linking Strategy

### Link Anchor Text Framework

**Primary Keywords** (High priority - 3-5 links)
```
<a href="/service-page/">primary keyword phrase</a>
```

**Long-tail Keywords** (Medium priority - 2-3 links)
```
<a href="/resource-page/">long-tail variation with context</a>
```

**Branded Links** (Low priority - as needed)
```
<a href="/tool-page/">click here to try tool</a>
```

### Content Linking Patterns

**Hub & Spoke Model:**
- Central pillar content (comprehensive guide)
- Cluster content (detailed topic pages)
- Cross-link: Pillar → Cluster (3-5 links)
- Cluster → Pillar (contextual links)

**Example Structure:**
```
Pillar: "Complete Guide to Project Management"
├── Cluster: "Agile Methodologies"
├── Cluster: "Team Collaboration Tools"
├── Cluster: "Resource Planning"
└── Cluster: "Risk Management"

Each cluster links back to pillar with anchor text
```

## Technical SEO Essentials

**Must-Haves:**
- [ ] Mobile-responsive design
- [ ] Page load speed < 3 seconds
- [ ] HTTPS enabled
- [ ] XML sitemap submitted
- [ ] Robots.txt configured
- [ ] Canonical tags on duplicate content
- [ ] Schema markup implemented
- [ ] 404 redirect to relevant pages

## SEO Content Checklist

- [ ] Keyword research completed and approved
- [ ] Primary keyword in title (first 60 characters)
- [ ] Primary keyword in H1
- [ ] Meta description written (155-160 chars)
- [ ] At least 3 H2 subheadings with variations
- [ ] Alt text on all images
- [ ] Internal links: 3-5 primary, 2-3 secondary
- [ ] Content length: 800+ words for competitive topics
- [ ] Mobile preview checked
- [ ] Page speed optimized
- [ ] Related content links added
- [ ] CTA included and prominent
